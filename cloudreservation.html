Cloud Reservation System for Lasertag Facilities

This client-facing site is a reservation system used by client lasertag facilities to sign customers in. It utilizes a USB mag stripe reader to read the ID number of a player card swiped upon arrival. The system collects and stores user information for game play, marketing, and player tracking. The reservation system has a customer-facing kiosk component as well as an employee-facing admin component.

Employees at the lasertag facility use the admin portion of the reservation system, pictured below, to register a player for a specific session. When a player card is swiped into the system by an employee, the card number is saved to the database along with session information. Javascript is used to allow the employee to navigate between session pages and perform edits to the player file. Once a game is initiated by an employee, all player information is sent to the lasertag game computer.

Players then swipe the card at a kiosk to enter their personal information and agree to terms of service. When that happens, the kiosk queries the database to verify that the card is registered. As long as it is, it permits the customer to complete the sign-in process. If the card is not registered, it will display an error.
