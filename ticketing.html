TICKETING SYSTEM

The Ticketing System is used internally to document and track customer technical and systems issues. When a ticket is entered, the priority assigned by the employee will initiate a series of PHP functions to set alerts and automated follow-up reminders.

Every time the system is accessed by an employee, the backend code looks at all open tickets and updates their status in real time. If tickets have exceeded their SLA, an automated alert is generated. The alerts can be manually edited by an employee, however manual changes are tracked for auditing purposes. E-mail status updates are automatically generated and sent to clients based on ticket priority.
